# AGENTS.md

## What This Project Is

Wayfinder is a Python 3.11+ implementation of the Wayfinder Interaction Protocol
(WIP): a Unix-style command protocol for representing goals, recommendations,
actions, updates, and append-only event history.

Current implementation status: the protocol core library exists under
`src/wayfinder/core/`. The `wayfinder` CLI entry point is intentionally still a
Phase 3 stub and exits with "not yet implemented"; do not expect examples from
`wayfinder-cli-user-guide.md` to run yet.

## Important Paths

- `src/wayfinder/core/`: implemented protocol core: canonical JSON, hash chain,
  event log, reducer, idempotency, validation, artifacts, locks, freshness, and
  update mapping.
- `src/wayfinder/cli.py`: current CLI stub.
- `schemas/`: source-controlled JSON Schema contracts for WIP objects. Treat
  these as protocol source, not generated output.
- `tests/`: pytest suite for implemented behavior.
- `wayfinder-interaction-protocol-v0.1.md`: canonical protocol spec.
- `wayfinder-cli-user-guide.md`: canonical guide for the intended CLI UX.
- `PLAN.md`: phased implementation plan.
- `hooks/copy_canonical_docs.py`: MkDocs hook that copies canonical root docs
  into `docs/` during site builds.
- `.github/workflows/`: CI and GitHub Pages publication.

The public repository and Pages configuration are recorded in `mkdocs.yml`.

## Setup and Commands

Prefer `uv` when available.

- Install dev/docs deps: `make install`
- Fast local check: `make fast`
- One focused pytest invocation: `make single TEST=tests/core/test_reducer.py`
- CLI stub smoke test: `make cli-smoke`
- Full tests: `make test`
- Coverage: `make coverage`
- Lint and format check: `make lint`
- Auto-format: `make format`
- Type check: `make typecheck`
- Security checks: `make security`
- Complexity checks: `make complexity`
- Docs build: `make docs-build`
- Docs serve: `make docs-serve`
- Full local gate: `make check`

`make docs-lint` uses `npx markdownlint-cli2` and `lychee`; it may need tools or
network setup beyond Python dependencies. `make security` runs `pip-audit`.

## Test Guidance

Pytest markers are defined in `pyproject.toml`:

- `conformance`: Appendix B conformance vectors, planned for Phase 3.
- `live`: tests requiring network or API keys.

For normal agent work, start with `make fast` or a focused
`uv run pytest path`. Run `make check` before handing off broader changes.

## Glossary

- WIP: Wayfinder Interaction Protocol.
- Goal: the durable user objective being advanced.
- Recommendation: the next proposed action or response from a wayfinder.
- Action: an executable or otherwise concrete step within a recommendation.
- Event log: append-only JSONL history; this is the canonical visible protocol
  state.
- Status: reduced view of a goal reconstructed from the event log.
- Reducer: deterministic replay logic that turns events into status.
- Freshness: checks that a recommendation is still valid for the current event
  log head, lease, and claim state.
- Idempotency: create/update replay protection based on canonical request
  hashes.
- Canonical bytes: RFC 8785 JSON bytes after removing only transport-only
  `request_id`.
- Hash chain: per-event integrity chain over canonical event payloads.

## Gotchas

- The event log is the canonical visible protocol state.
- Canonical JSON means RFC 8785 bytes after removing only transport-only
  `request_id`.
- Reducer behavior must be deterministic and must not consult the clock.
- Snapshots may speed reads, but must not replace the event log in WIP v0.1.
- CLI examples in the guide are aspirational until Phase 3 lands.
- `make clean` deletes generated docs copies, site output, coverage, and local
  caches.
